# [Git] 깃 개념 및 기초 명령어

## 왜 깃을 써야 할까?
VCS(버전 관리 시스템)의 일종. 개발하다가 언제든 과거로 돌아갈 수 있는 **'세이브 포인트'**를 만들어줌. 브랜치 파서 여러 버전으로 안전하게 테스트해 보기도 좋음. 
당장 명령어 다 못 외워도 흐름 파악하는 게 우선!

<br>

## CLI vs GUI
- **CLI (터미널 명령어):** 뼈대 잡고 깃 기능 100% 활용하려면 CLI로 손에 익히는 게 좋음. (명령어 까먹으면 AI한테 물어보면 됨)
- **GUI (소스트리 등):** 시각화 도구. 커밋 히스토리나 전체적인 흐름 볼 때 꿀.
> CLI로 명령 내리고 GUI로 확인하는 조합 추천.

<br>

## 처음 깔고 무조건 해야 하는 전역 설정
컴퓨터에 내 정보 등록하고 기본 브랜치 이름 세팅하기.
```bash
git config --global user.name "내 이름"
git config --global user.email "내 이메일"
git config --global init.defaultBranch main # master 대신 main 쓰기
```

<br>

## 깃 기본 명령어
- `git init` : "여기서부터 깃 쓸게" 선언. (숨김 파일로 `.git` 생김)
- `git status` : 현재 상태 확인. 
  - *Untracked files*: 깃이 아직 추적 안 하는 완전 쌩 뉴비 파일들.
> **※ 주의사항:** 깃은 공백이나 쉼표 하나만 쳐도 '변경'으로 귀신같이 감지함. 오타 조심할 것.

<br>

## 깃에 절대 올리면 안 되는 애들 (`.gitignore`)
비밀번호, API Key 같은 보안 정보나 무거운 라이브러리(node_modules 등), 빌드된 파일은 깃이 무시하도록 세팅해야 함.
`.gitignore` 파일 만들어서 블랙리스트 적어두면 됨. (양식은 AI한테 짜달라고 하면 편함)

<br>

## 세이브 포인트 만들기 (Commit)
- **`git add <파일명>`** : 타임캡슐에 담기 (보통 `git add .`으로 전체 다 담음)
- **`git commit -m "메시지"`** : 타임캡슐 묻기 (세이브 포인트 생성)
- **`git commit -am "메시지"`** : Untracked 파일 없으면 add랑 commit 한 번에 쌉가능.

**기록 확인하기**
- `git log`, `git diff`로 확인 가능하지만 이건 소스트리 같은 GUI로 보는 게 훨씬 빠르고 편함.

<br>

## 과거로 돌아가기 (Reset / Revert / Checkout)

### 1. Reset (기록 삭제, 찐 과거로)
특정 시점으로 아예 되감기. **그 이후의 미래(커밋)는 싹 다 날아감.**
```bash
git reset --hard <돌아갈 커밋해시>
```

### 2. Revert (기록 유지, 흑역사 덮기)
과거의 작업을 반대로 실행하는 **'새로운 커밋'**을 추가함. 실무에서 기존 기록 안 건드리고 원상복구할 때 많이 씀.
```bash
git revert <되돌릴 커밋해시>
```
**Revert 할 때 충돌(Conflict) 나면?**
내가 되돌리려는 커밋 이후에 또 수정된 파일이 있으면 깃이 뇌정지 옴.
1. 직접 파일 수정해서 찐막 상태 정해줌
2. `git add <파일>` (또는 삭제할 거면 `git rm <파일>`)
3. `git revert --continue` 입력해서 마무리
> * 쫄리면 `--abort`로 아예 취소하거나, 여러 개 하는 중이었으면 `--skip`으로 건너뛰기도 가능.
> * `A..B` 처럼 범위 지정하면 B(최신)부터 A 다음 커밋까지 역순으로 차례대로 Revert 됨.

### 3. Checkout (과거 구경 다녀오기)
기록 안 지우고, 새 커밋도 안 만들고 진짜 특정 시점의 코드로 잠깐 뷰(View)만 이동하는 것. 거기서 새 브랜치 파서 다른 차원 열 수도 있음.
